## Reading

- Performance is **green** (1.0 desktop, 0.92 mobile). The script payload that ships is heavier than the target but the LCP / CLS / INP signals still clear the thresholds.
- Accessibility holds at **0.97 / 0.97** — three minor findings short of 1.0. Axe-core scan landing in PR-4 will surface specifics.
- Best Practices and SEO both at **0.96 / 0.92** — within the existing `warn` thresholds; SEO short of the PRD's 1.0 aspiration likely due to `meta description` length or robots-meta detail.
- **Script size 386 KiB is the load-bearing finding.** The current `lighthouserc.yml` enforces a 200 KiB warn ceiling — every PR has been warning. The right move per gap-analysis plan §PR-3 is **not to silently raise the ceiling**; instead capture the overrun, set a new `warn` ceiling at the measured value to suppress noise without erasing the obligation, and open a follow-up `perf/budget-overrun` ROADMAP line.

## Plan landing in this PR

- Promote category assertions in both configs from `warn` to `error` at thresholds the current measurement clears.
- Bump the script-size `warn` ceiling from 200 KiB to 400 KiB (slightly above measured 377 KiB) so the gate suppresses noise; **do not** make it `error` until the overrun is resolved.
- Add a ROADMAP follow-up `[ ] Script-bundle overrun (377 KiB > 200 KiB target) — investigate top chunks + reduce`. This is the explicit, tracked follow-up rather than silent ceiling growth.

## Path back to 150 KiB

The 820 KiB unhydrated chunk strongly suggests three.js + R3F + drei landing in a chunk the Next.js preloader pre-fetches even when the import is `ssr: false`. Likely investigations:

1. Audit `next/dynamic` boundaries — confirm both `AgentGraphClient` and the disabled `WandererCraneClient` are truly lazy.
2. Check whether `<head>` `<link rel="preload">` includes the R3F chunk (Next 16 default).
3. Consider replacing R3F with raw Three.js for the AgentGraph scene (matches the pattern the Wanderer already uses) — would shed `@react-three/fiber` + `@react-three/drei` from the dependency tree.
4. The disabled `framer-motion` + `gsap` + `lucide-react` declared in `package.json` are not currently imported, but verify via `pnpm why` they don't have transitive consumers.

PR-6 of the gap-analysis plan removes the three unused deps (`framer-motion`, `gsap`, `lucide-react`). Investigation 3 is a P3 candidate for a future feature PR.
