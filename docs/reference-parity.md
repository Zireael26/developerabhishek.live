# Reference Parity Register

Accepted deviations from the original reference implementation:

- Canonical domain is `akaushik.org`; legacy `developerabhishek.live` references are historical only.
- Home, work, and writing surfaces use real MDX content instead of reference placeholders.
- Work cards and detail hero bands may use committed HyperFrames videos where the reference used static placeholders.
- Mobile hero ordering favors the live content hierarchy over the reference's desktop-first sequence.
- Typography is served through `next/font` rather than the reference's direct font loading.
- Work detail pages are richer than the reference cards, with full case-study context and Markdown alternates.
